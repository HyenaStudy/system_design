# Ch_1. 사용자 수에 따른 규모 확장성
## 데이터베이스
### RDBMS
- MySQL : 읽기 여부 중요, 직관적인 쿼리, 단일 프로세스 내 멀티 스레드
- PostgreSQL : 쓰기 작업 최적, 대규모, 복잡한 트랜잭션 처리, 배열과 JSON 등 다양한 데이터 타입, 멀티 프로세스
### NoSQL
- Redis : 인메모리 기반, Pub/Sub, 정렬 집합과 스트림 등 다양한 데이터 타입, 캐시 및 MQ 활용 가능성
- MongoDB : 문서형, JSON 형태, 샤딩 기반 수평 확장 용이
## 스케일링
- 스케일 업(수직적 규모 확장) : 돈 바르기
- 스케일 아웃(수평적 규모 확장) : LB, DB 다중화
## 캐시
- 읽기 주도형 전략? 캐시 갱신 타이밍(캐시미스 이슈), DB 일관성 준수
## 그 외(etc)
- 세션 데이터 웹 계층 분리(무상태성 보장), 메세지 큐, DB 레플리케이션 & 샤딩, 로그 및 메트릭 수집, CDN, 데이터 센터 등등...
